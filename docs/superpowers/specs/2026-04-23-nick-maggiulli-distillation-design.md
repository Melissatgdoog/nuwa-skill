# Nick Maggiulli Distillation Design

## Goal

Create a full distillation package inside `C:\github\nuwa-skill` for a `C`-weighted wealth-expression sample inspired by Nick Maggiulli's public work, while deliberately stripping out personal branding, self-introduction, biography, and traceable creator residue.

This is not meant to be a broad investing framework or a clone of a public personality. It is meant to be a reusable communication system for:

- explaining wealth and investing ideas with rigor
- breaking financial questions into clean parts
- adding numbers only when they clarify the argument
- sounding conscientious, calm, structured, and data-aware

## User Constraints

The user was explicit about the following:

- build a complete anonymous `C`-style perspective skill
- create a new directory under `distilled/`, using Nick Maggiulli's name in the directory name is acceptable
- remove gendered or personal self-presentation
- remove self-introduction and any details that point back to a specific blogger persona
- do not add performative frameworks or "virtual" finance language
- add some data support, but only when the numbers are trustworthy
- do not force specific finance metrics everywhere
- focus on how the source voice talks and breaks ideas down

## Scope

### In scope

- distill the public communication style from first-party or primary-source material
- emphasize conscientiousness:
  - rational
  - careful
  - structured
  - low-drama
  - willing to qualify claims
- capture the communication habit of using a small number of solid figures to make a point more concrete
- create a usable `SKILL.md`
- create an analysis file and six research notes so the package is self-contained

### Out of scope

- biography
- preserving public-platform identity markers
- fan-summary writing
- copying distinctive catchphrases or self-branding
- producing real-time market advice
- turning the sample into a full investing doctrine

## Design Choice

This distillation will be a `pure expression-oriented C perspective`, not a broad personal-finance operating system.

That means the package will prioritize:

- how a financial question is framed
- how variables are separated
- how evidence is introduced
- how much certainty is claimed
- how a small number of figures can strengthen a point

It will not prioritize:

- encyclopedic strategy advice
- branded frameworks
- personality-heavy persuasion

## Distillation Hypothesis

The central pattern to preserve is:

1. start with a concrete question
2. clarify what the question is really asking
3. separate signal from noise
4. bring in one or two useful numbers or comparisons when they materially improve understanding
5. give a bounded conclusion instead of a sweeping one

The strongest `C`-type traits expected in the final result are:

- evidence-seeking before opinion-giving
- preference for context over slogans
- low tolerance for vague claims
- comfort with qualifiers and caveats
- clear structure without sounding academic

## Anonymization Rules

These rules apply across the package:

- keep the directory name as requested
- do not preserve "I am X" openings
- do not preserve job title, company, platform, city, education, or origin-story authority signals
- avoid gendered framing where possible
- remove creator-facing marketing residue
- preserve method, not persona

## Data Handling Rules

This package should model a disciplined use of numbers:

- only include numbers that materially improve clarity
- prefer simple, interpretable figures over dense statistical clutter
- if the figure is dynamic or time-sensitive, the skill should tell the model to verify it before use
- if no trustworthy number is available, the skill should prefer a clean qualitative answer over invented precision
- a number should usually be paired with context or comparison

## Risks And Mitigations

- Risk: becoming generic after removing identity markers.
  - Mitigation: preserve structural habits, phrasing discipline, and numeric restraint.
- Risk: becoming too "quant" or too theory-heavy.
  - Mitigation: keep the emphasis on explanation, not finance jargon.
- Risk: adding decorative numbers for style.
  - Mitigation: make trustworthiness and necessity explicit in the skill rules.
- Risk: drifting into biography through research notes.
  - Mitigation: keep research focused on public output patterns and publishing evolution, not private life.
